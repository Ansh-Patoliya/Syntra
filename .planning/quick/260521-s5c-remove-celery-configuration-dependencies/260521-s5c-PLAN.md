# Quick Task: Remove Celery configuration, dependencies, and code, and run seating allocation synchronously

This quick task removes Celery background job infrastructure and configuration from the backend, making seating allocation run synchronously.

## Verification Criteria
- [ ] Celery imports and configurations are completely removed.
- [ ] The seating allocation API endpoint (`/api/organizer/hackathons/<id>/allocate-seats/`) runs synchronously and updates the database successfully.
- [ ] Django starts up normally without any Celery dependencies.
- [ ] Project unit tests run and pass without errors.

## must_haves
- Remove celery configurations from settings.py and init files.
- Remove celery-related dependencies from requirements.txt.
- Remove backend/syntra/celery.py and backend/organizer/tasks.py.
- Seating allocation in api_views.py must execute synchronously.
