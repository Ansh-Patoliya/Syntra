# Quick Task: Login and Signup Error Highlighting

This quick task implements premium SaaS error highlighting for credentials validation failures on the login and signup forms.

## Verification Criteria
- [ ] Submitting invalid credentials shows the error alert banner inside the card.
- [ ] The error banner has a soft red background, red left border, and red text (styled according to the design system).
- [ ] On credentials failure, the login/signup card performs a smooth horizontal shake micro-animation.
- [ ] Submitting invalid credentials multiple times in a row triggers the shake animation on each failure.

## must_haves
- Set classes of the API message box to `message message--error` or `message message--success` in `auth_api.js`.
- Define `@keyframes shake` and `.shake` in `design-system.css`.
- Trigger the `.shake` class on `.auth-card` in `auth_api.js` on error.
