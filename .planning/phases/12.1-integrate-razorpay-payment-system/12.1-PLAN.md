# Phase 12.1: Integrate Razorpay payment system for hackathons - Plan

This document outlines the step-by-step execution plan for integrating the Razorpay payment system based on the phase context.

## Prerequisites
- [ ] `razorpay` dependency added to `backend/requirements.txt` and installed.
- [ ] `RAZORPAY_KEY_ID` and `RAZORPAY_KEY_SECRET` read from the environment variables in `syntra/settings.py`.

## Step 1: Model Updates
- [ ] **`organizer.models.Hackathon`**:
  - Add `is_paid = models.BooleanField(default=False)`
  - Add `fee_type = models.CharField(max_length=20, choices=[('team', 'Team Wise'), ('participant', 'Participant Wise')], null=True, blank=True)`
  - Add `fee_amount = models.DecimalField(max_digits=10, decimal_places=2, null=True, blank=True)`
- [ ] **`participant.models.Payment`** (New Model):
  - Fields: `team` (ForeignKey), `user` (ForeignKey), `razorpay_order_id` (CharField), `razorpay_payment_id` (CharField), `razorpay_signature` (CharField), `amount` (DecimalField), `status` (CharField choices: pending, successful, failed).
- [ ] Make and run migrations.

## Step 2: Organizer Configuration
- [ ] **`organizer.forms.HackathonForm`**: 
  - Add `is_paid`, `fee_type`, and `fee_amount` fields.
  - Implement frontend logic to show `fee_type` and `fee_amount` only when `is_paid` is checked.

## Step 3: Backend Payment Service
- [ ] **`participant.payment_services.py`**:
  - Initialize Razorpay client.
  - Create function `create_razorpay_order(amount, team, user)` which hits Razorpay API, generates an order, saves it in `Payment` model as `pending`, and returns the order payload.
  - Create function `verify_razorpay_signature(razorpay_order_id, razorpay_payment_id, razorpay_signature)` which calls `client.utility.verify_payment_signature(...)`.

## Step 4: Participant Registration Logic
- [ ] **`participant.views.py`**: 
  - Update team registration finalization. If `hackathon.is_paid` is true:
    - Calculate amount: if `fee_type == 'team'` then `amount = fee_amount`. If `fee_type == 'participant'` then `amount = fee_amount * team.members.count()`.
    - Create order via `create_razorpay_order`.
    - Redirect to a new payment view `PaymentCheckoutView` passing the `order_id`.
  - Create `PaymentCheckoutView` to render the Razorpay modal.
  - Create `PaymentVerifyView` to handle the POST callback from Razorpay Checkout, verify the signature, mark payment as `successful`, and finalize the registration (generate QR code, activate team).

## Step 5: Frontend Integration
- [ ] **`templates/participant/payment_checkout.html`**:
  - Inject the `<script src="https://checkout.razorpay.com/v1/checkout.js"></script>`.
  - Pass the order details (amount, order_id, razorpay_key_id) to the JS.
  - Setup the `options.handler` callback to POST to `PaymentVerifyView`.
